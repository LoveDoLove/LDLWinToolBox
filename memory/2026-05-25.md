## 关键决策

### 决策 1：记忆系统的三层结构

```
MEMORY.md（长期背景）
    ↓
memory/tasks.md（跨对话任务追踪）
    ↓
memory/YYYY-MM-DD.md（每日详细日志）
```

**理由：** 
- 分层便于不同时间尺度的信息管理
- MEMORY.md 适合项目初期化和一次性规范更新
- tasks.md 适合跨对话的进度追踪
- 每日日志适合决策历史和学习记录

### 决策 2：严格遵循现有项目规范

在编写 AGENTS.md 和 MEMORY.md 时，完全基于现有的：
- `ANALYSIS.md`（技术分析）
- `PROMPT_GUIDE.md`（使用指南）
- `LDLWinToolBox.bat`（源代码）

**理由：** 避免创造新的规范，而是提炼和整理现有的隐含约定

### 决策 3：技能包从开源克隆，不自行编写

在 AGENTS.md 和 MEMORY.md 中明确禁止自行编写技能包，必须从 GitHub 或 Skills.sh 克隆

**理由：**
- 遵循 DRY（Don't Repeat Yourself）原则
- 开源技能包通常经过社区验证，质量更高
- 便于文档和社区支持

## 技术笔记

### 项目文件结构回顾

```
LDLWinToolBox/
├── README.md              # 项目首页
├── ANALYSIS.md            # 技术分析（7 个部分）
├── PROMPT_GUIDE.md        # 用户和开发者指南
├── LDLWinToolBox.bat      # 主程序（菜单驱动）
├── AGENTS.md              # [新] AI Agent 规范
├── MEMORY.md              # [新] 长期记忆库
├── memory/                # [新] 日志目录
│   ├── tasks.md           # 任务追踪
│   └── 2026-05-25.md      # 今日日志（可重复）
└── .agents/               # [新] Agent 基础设施
    └── skills/            # 技能包库（待填充）
        └── karpathy-guidelines/  # 样例技能包
```

### Batch 脚本的关键特性

项目通过以下技术方案实现高质量的 Windows 工具：

1. **延迟变量展开** → `setlocal EnableDelayedExpansion` 和 `!VAR!` 语法
2. **权限双层提升** → `cacls.exe` 检查 + PowerShell `RunAs`
3. **时间戳日志** → PowerShell `Get-Date -Format yyMMddHHmmss`
4. **动态空间计算** → WMI `Win32_LogicalDisk` 查询
5. **菜单驱动交互** → 标签跳转（`:label` + `goto label`）

建议在将来的维护中统一这两个部分。
